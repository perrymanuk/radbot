# E2E Cost Optimization Research

Research common solutions for LLM cost optimization based on findings from a recent `/e2e-test` run. Uses web searches to find best practices, official documentation, and community solutions.

## Pre-requisite Check

This skill is designed to run directly after `/e2e-test`, using that run's results. Verify the expected artifacts exist:

1. Check `reports/e2e-cost-analysis.md` exists (generated by the cost analyst teammate)
2. Check `reports/e2e-session-usage.json` exists (session usage snapshot)
3. Check `reports/e2e-performance-review.md` exists (performance findings may inform research)

If the cost analysis report doesn't exist, check if `reports/e2e-session-usage.json` exists as a fallback — the research can proceed with raw usage data. If neither exists, stop and tell the user: "Run `/e2e-test` first to generate cost data for analysis."

## Flow

### 1. Parse E2E Run Findings

Read the artifacts from the most recent `/e2e-test` run:
- `reports/e2e-cost-analysis.md` — cost analyst's findings and recommendations
- `reports/e2e-session-usage.json` — raw per-agent token/cost data
- `reports/e2e-performance-review.md` — performance findings (may contain cost-related insights)

Extract:
- The top optimization recommendations from the cost analyst
- Per-agent cost breakdown and model usage
- Cache hit rates and token efficiency metrics
- Specific problem areas flagged by any teammate

Summarize the key findings to the user before spawning research agents.

### 2. Spawn Research Agents

Create a team and spawn 2-3 targeted research agents based on findings. Each agent should use WebSearch to find solutions, then write a report.

```
TeamCreate(team_name="e2e-cost-research", description="Cost optimization research team")
```

**Research Agent 1: Caching & API Optimization**
- `Agent(name="cache-researcher", team_name="e2e-cost-research", run_in_background=true)`
- Prompt: Read `.claude/skills/e2e-test-research/research-guide.md` for instructions. Research Google ADK context caching and Gemini API caching strategies. Focus on:
  - How ContextCacheConfig works in Google ADK (cache_intervals, ttl_seconds, min_tokens)
  - What exactly gets cached (system prompt only? conversation history prefix?)
  - Best practices for improving cache hit rates in multi-agent systems
  - Gemini API context caching documentation and pricing
  - Search queries: "Google ADK ContextCacheConfig", "Gemini API context caching best practices", "google adk cache_intervals tuning"
  - Write findings to `reports/e2e-research-caching.md`

**Research Agent 2: Prompt & Token Optimization**
- `Agent(name="prompt-researcher", team_name="e2e-cost-research", run_in_background=true)`
- Prompt: Read `.claude/skills/e2e-test-research/research-guide.md` for instructions. Research LLM prompt optimization and token reduction strategies. Focus on:
  - Reducing system prompt token count without losing capability
  - Managing conversation history growth in multi-turn sessions
  - Model selection strategies for routing/orchestrator agents
  - Gemini model comparison (pro vs flash) for routing tasks
  - Search queries: "LLM prompt optimization reduce tokens", "multi-agent orchestrator model selection", "gemini flash vs pro routing tasks", "conversation history management LLM cost"
  - Write findings to `reports/e2e-research-prompts.md`

**Research Agent 3: Architecture Patterns** (spawn only if findings indicate architecture issues like high transfer counts, conversation history bloat, or inefficient agent routing)
- `Agent(name="arch-researcher", team_name="e2e-cost-research", run_in_background=true)`
- Prompt: Read `.claude/skills/e2e-test-research/research-guide.md` for instructions. Research multi-agent architecture patterns for cost optimization. Focus on:
  - Google ADK session management and conversation pruning
  - Multi-agent routing patterns that minimize LLM calls
  - Techniques for reducing agent transfer overhead
  - Search queries: "Google ADK multi-agent cost optimization", "LLM agent routing without model calls", "ADK session history management"
  - Write findings to `reports/e2e-research-architecture.md`

### 3. Wait for Research Agents

Wait for all research agents to complete. You will be notified automatically.

### 4. Synthesize Optimization Plan

Read all research reports:
- `reports/e2e-research-caching.md`
- `reports/e2e-research-prompts.md`
- `reports/e2e-research-architecture.md` (if it exists)

Also re-read `reports/e2e-cost-analysis.md` for the original findings.

Generate a unified, research-backed optimization plan at `reports/e2e-optimization-plan.md`:

```markdown
# Cost Optimization Plan

## Current State
- Total e2e session cost: $X.XX
- Cost per test: $X.XX
- Top cost driver: <agent> at N% of total
- Cache hit rate: X.X%

## Research-Backed Recommendations

### 1. <Title> — Estimated savings: $X.XX/run (N%)
- **Problem**: <what the cost analysis found>
- **Research findings**: <what web research revealed, with source references>
- **Proposed implementation**:
  - File: `path/to/file.py`
  - Change: <specific code change>
- **Risk assessment**: <what could go wrong, mitigation>
- **Effort**: Low / Medium / High

### 2. ...

## Implementation Priority

| # | Change | Savings | Effort | Priority |
|---|--------|---------|--------|----------|
| 1 | ... | $X.XX | Low | Do first |
| 2 | ... | $X.XX | Medium | Do second |

## Sources
- [Source 1](url) — <what it contributed>
- [Source 2](url) — <what it contributed>
```

### 5. Teardown

Clean up the team:
```
TeamDelete(team_name="e2e-cost-research")
```

Report the optimization plan location to the user and summarize the top 3 recommendations.

## Important Notes

- This skill is research-only — it does NOT make code changes
- Always ground recommendations in both the actual cost data AND external research
- Prefer official Google/ADK documentation over blog posts
- Each recommendation must include a specific file path and proposed change
- Quantify estimated savings for every recommendation
- If research contradicts the cost analyst's recommendations, note the disagreement and explain why
